# Auto-Scholar 论文导入功能 - 迭代方案

## 📋 需求理解

### 核心需求
在 Auto-Scholar 模块的"收藏列表"中，当用户点击某篇论文的"导入论文库"按钮时：
1. 系统自动下载该论文的 PDF 文件（从 Arxiv）
2. 调用现有的论文分析能力（与上传论文相同的流程）
3. 分析完成后，论文加入主页数据库并在主页显示

### 功能目标
- **打通 Auto-Scholar 和主论文库**：让用户可以将感兴趣的 Arxiv 论文一键导入到主论文库
- **复用现有分析能力**：使用与手动上传相同的分析流程（总结、思维导图、标签、RAG）
- **自动化体验**：用户无需手动下载 PDF，系统自动完成


## ⚠️ 潜在风险与应对

### 风险 1: PDF 下载失败

**原因**：
- Arxiv 服务器不可用
- 网络连接问题
- PDF 文件不存在

**应对**：
- 添加重试机制（3次重试，间隔2秒）
- 显示友好的错误提示
- 提供手动下载链接

### 风险 2: 重复导入

**原因**：
- 用户多次点击"导入"按钮
- 同一篇论文已存在于主论文库

**应对**：
- 检查 `is_imported` 标志
- 检查 Paper 表中是否存在同名论文
- 提供"更新"或"跳过"选项

### 风险 3: 处理时间过长

**原因**：
- LLM API 响应慢
- PDF 文件较大

**应对**：
- 显示详细的进度条
- 添加"后台处理"选项（未来）
- 设置超时机制

### 风险 4: 存储空间不足

**原因**：
- PDF 文件较大
- 磁盘空间不足

**应对**：
- 检查磁盘空间
- 提供清理功能
- 限制单个 PDF 大小（如 50MB）
