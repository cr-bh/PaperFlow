# 更新日志 v1.4.0 - 论文发表数据统计分析与可视化功能

## 📅 更新日期
2026-02-13

## 🎯 更新概述

本次更新为 Auto-Scholar 功能添加了全面的论文发表数据统计分析与可视化功能，包括：
- **Tab 4: 统计分析** - 基于已评分论文的质量分析（扩充）
- **Tab 5: 发表趋势** - 基于Arxiv API实时抓取的轻量级趋势分析（新增）

## ✨ 新增功能

### 1. Tab 5: 发表趋势（轻量级）

#### 核心特性
- **实时数据抓取**：从Arxiv API实时抓取论文数据，不依赖评分系统
- **时间范围选择**：支持自定义时间范围（最多30天）
- **完整趋势分析**：查看完整的论文发表趋势，不受评分筛选限制

#### 功能模块

##### 📅 时间趋势分析
- 每日新增论文数量趋势图（折线图）
- 统计指标：总论文数、日均论文数、单日最高
- 支持按周聚合统计

##### 🔍 关键词分析
- 关键词论文数量分布（柱状图）
- 关键词类别占比（饼图：核心 vs 前沿）
- **多关键词趋势对比**（P2阶段）：
  - 支持选择最多5个关键词进行时间趋势对比
  - 多条折线图展示不同关键词的热度变化

##### 🔥 热力图分析
- 关键词×时间热力图
- 直观展示关键词在不同时间的热度分布
- 自动限制显示前20个关键词（避免图表过大）

### 2. Tab 4: 统计分析（扩充）

#### 新增子Tab结构
- **📊 分数分析**（保留原有功能）
- **🏆 顶会顶刊**（新增）
- **🏛️ 知名机构**（新增）
- **🔬 交叉分析**（P2阶段新增）

#### 新增功能模块

##### 🏆 顶会顶刊分析
- 顶会顶刊论文数量排行（横向柱状图）
- 显示每个会议/期刊的论文数量和平均分
- 支持最低分数筛选
- 详细数据展开查看

##### 🏛️ 知名机构分析
- 知名机构论文数量排行（横向柱状图）
- 显示每个机构的论文数量和平均分
- 自动解析 institutions JSON 字段
- 详细数据展开查看

##### 🔬 交叉分析（P2阶段）
- **会议×分数交叉分析**：
  - 分组堆叠柱状图
  - 展示各会议在不同分数等级（S/A/B/C）的论文分布
  - 支持选择前N个会议进行分析

- **机构×分数交叉分析**：
  - 分组堆叠柱状图
  - 展示各机构在不同分数等级的论文分布
  - 支持选择前N个机构进行分析

## 🏗️ 技术架构

### 新增文件

#### 1. `services/trend_analyzer.py`
轻量级趋势分析器，从Arxiv API实时抓取数据

**核心方法**：
- `fetch_papers_by_date_range()` - 从Arxiv API抓取指定日期范围的论文
- `get_daily_paper_count()` - 统计每日论文数量
- `get_weekly_aggregation()` - 按周聚合统计
- `get_keyword_distribution()` - 统计各关键词的论文数量
- `get_keyword_category_ratio()` - 统计关键词类别占比
- `get_keyword_trend()` - 获取单个关键词的时间趋势
- `get_keyword_cooccurrence()` - 获取关键词共现矩阵
- `get_keyword_time_heatmap()` - 获取关键词×时间热力图数据

**特点**：
- 不依赖数据库，直接从Arxiv API抓取
- 不包含 score, venue, institutions（轻量化）
- 支持日期范围查询和关键词筛选

#### 2. `services/quality_analyzer.py`
质量分析器，基于已评分论文（ArxivPaper表）

**核心方法**：
- `get_score_distribution()` - 获取分数分布（S/A/B/C级）
- `get_score_statistics()` - 获取分数统计（平均分、中位数等）
- `get_venue_distribution()` - 获取顶会顶刊分布和平均分
- `get_institution_distribution()` - 获取知名机构分布和平均分
- `get_author_productivity()` - 获取高产作者排行
- `get_venue_score_matrix()` - 获取会议×分数交叉分析
- `get_institution_score_matrix()` - 获取机构×分数交叉分析

**特点**：
- 基于数据库查询，数据完整
- 包含评分、会议、机构等完整信息
- 支持最低分数筛选和Top N限制

## 📊 数据流设计

### Tab 5: 发表趋势（轻量级）
```
用户选择时间范围 → 点击"开始分析"
    ↓
从Arxiv API抓取数据（实时）
    ↓
缓存到 session_state
    ↓
显示分析结果（时间趋势、关键词分析、热力图）
```

### Tab 4: 统计分析（质量）
```
从数据库获取已评分论文
    ↓
QualityAnalyzer 分析
    ↓
显示分析结果（分数、会议、机构、交叉分析）
```

## 🎨 可视化设计

### 图表类型
- **折线图**：时间趋势、多关键词对比
- **柱状图**：关键词分布、分数统计
- **横向柱状图**：会议排行、机构排行
- **饼图**：分数等级分布、关键词类别占比
- **直方图**：分数详细分布
- **热力图**：关键词×时间
- **分组堆叠柱状图**：会议×分数、机构×分数

### 配色方案
- S级：`#e74c3c`（红色）
- A级：`#3498db`（蓝色）
- B级：`#95a5a6`（灰色）
- C级：`#bdc3c7`（浅灰）
- 会议：`#9b59b6`（紫色）
- 机构：`#16a085`（青色）

## 🔧 性能优化

### 1. 查询范围限制
- 发表趋势分析：最多30天
- 统计分析：最多500篇论文
- 热力图：最多20个关键词

### 2. 缓存机制
- 使用 `st.session_state` 缓存抓取的论文数据
- 避免重复抓取，提升用户体验

### 3. 数据聚合
- 当数据点过多时，支持按周聚合
- 限制Top N显示，避免图表过大

## 📝 使用说明

### Tab 5: 发表趋势

1. **选择时间范围**：
   - 开始日期和结束日期（最多30天）
   - 系统会自动调整超出范围的日期

2. **点击"开始分析"**：
   - 系统从Arxiv API实时抓取数据
   - 显示进度提示
   - 抓取完成后显示论文数量

3. **查看分析结果**：
   - **时间趋势**：查看每日论文数量变化
   - **关键词分析**：查看关键词分布和趋势对比
   - **热力图**：查看关键词×时间的热度分布

### Tab 4: 统计分析

1. **分数分析**：
   - 查看论文等级分布（饼图、柱状图）
   - 查看统计指标（平均分、中位数、最高分、最低分）
   - 查看分数详细分布（直方图）
   - 查看最高分论文 Top 5

2. **顶会顶刊**：
   - 查看顶会顶刊论文数量排行
   - 每个会议显示论文数量和平均分
   - 展开查看详细数据

3. **知名机构**：
   - 查看知名机构论文数量排行
   - 每个机构显示论文数量和平均分
   - 展开查看详细数据

4. **交叉分析**：
   - 选择分析类型（会议×分数 或 机构×分数）
   - 查看分组堆叠柱状图
   - 展开查看详细数据

## ⚠️ 注意事项

### 1. 数据源差异
- **Tab 5（发表趋势）**：从Arxiv API实时抓取，数据量大但不包含评分信息
- **Tab 4（统计分析）**：从数据库读取已评分论文，数据完整但数量较少

### 2. 查询限制
- 发表趋势分析最多查询30天，避免API超时
- 关键词热力图最多显示20个关键词，避免图表过大

### 3. 性能考虑
- 首次抓取数据可能需要较长时间（取决于论文数量）
- 数据会缓存到 session_state，切换子Tab不会重新抓取
- 刷新页面会清空缓存，需要重新抓取
