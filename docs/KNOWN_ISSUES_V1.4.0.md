## ⚠️ 遗留问题

### 问题 #1: Tab 5 关键词选择首次跳转问题

**优先级**: 🟡 中等

**问题描述**:
在 Tab 5（发表趋势）→ "关键词分析"子Tab → "关键词时间趋势对比"模块中：
- 用户**第一次**选择/修改关键词后，页面会跳转回"时间趋势"子Tab
- 用户需要手动切回"关键词分析"子Tab，可以看到折线图已正确更新
- **第二次及之后**的修改不会跳转，工作正常

**复现步骤**:
1. 进入 Auto-Scholar → 发表趋势 Tab
2. 选择时间范围，点击"开始分析"
3. 切换到"关键词分析"子Tab
4. 滚动到"关键词时间趋势对比"部分
5. 在下拉框中选择/取消选择关键词（第一次操作）
6. 观察：页面跳转回"时间趋势"子Tab

**根本原因**:
- Streamlit 的 `st.tabs()` 组件在页面完全重新渲染时会重置到第一个Tab
- `st.multiselect` 首次交互时，`st.session_state[key]` 从"不存在"变为"有值"
- Streamlit 认为这是重大状态变化，触发页面完全重新渲染
- 导致 Tab 被重置

**已尝试的解决方案**:
1. ❌ 添加 `key` 参数
2. ❌ 使用 `st.session_state` 管理默认值
3. ❌ 移除 `default` 参数
4. ❌ 提前初始化状态（在数据抓取完成时）

详细的尝试记录见 [DEBUG_LOG_V1.4.0.md](DEBUG_LOG_V1.4.0.md#问题-4-tab跳转问题)

**影响评估**:
- **严重程度**: 🟡 低-中等
- **频率**: 仅第一次修改时发生（每次数据抓取后的第一次）
- **用户体验影响**:
  - ➖ 需要手动切回Tab（轻微不便）
  - ➕ 折线图功能正常，数据更新正确
  - ➕ 第二次之后工作正常
- **功能完整性**: ✅ 不影响核心功能

**可能的解决方案**:

#### 方案 A: 使用 `st.fragment` (推荐)
**要求**: Streamlit >= 1.18.0

```python
@st.fragment
def keyword_comparison_widget(papers, keywords):
    selected_keywords = st.multiselect(
        "选择要对比的关键词（最多5个）",
        options=keywords,
        max_selections=5,
        key="trend_keyword_comparison"
    )
    # ... 绘制折线图
```

**优点**:
- 隔离组件的重新运行，不影响父组件
- 官方推荐的解决方案

**缺点**:
- 需要 Streamlit 版本 >= 1.18.0
- 需要重构代码结构

**实施难度**: 🟡 中等

---

#### 方案 B: 使用 `st.experimental_rerun` 控制
**思路**: 在首次初始化后立即 rerun，确保状态稳定

```python
if "trend_keyword_comparison" not in st.session_state:
    st.session_state.trend_keyword_comparison = keywords[:min(3, len(keywords))]
    st.experimental_rerun()
```

**优点**:
- 不需要升级 Streamlit 版本
- 实施简单

**缺点**:
- 会导致额外的页面刷新
- 可能影响性能
- 不是最佳实践

**实施难度**: 🟢 简单

---

#### 方案 C: 改用单选 + 多选按钮
**思路**: 使用 `st.checkbox` 或 `st.button` 代替 `st.multiselect`

```python
st.write("选择要对比的关键词（最多5个）：")
selected_keywords = []
for keyword in keywords:
    if st.checkbox(keyword, key=f"kw_{keyword}"):
        selected_keywords.append(keyword)
        if len(selected_keywords) >= 5:
            break
```

**优点**:
- 避免 `st.multiselect` 的状态管理问题
- 用户体验可能更直观

**缺点**:
- UI 占用空间较大
- 用户体验改变较大

**实施难度**: 🟢 简单

---

#### 方案 D: 接受现状，添加用户提示
**思路**: 在 UI 上添加提示信息，告知用户这是正常行为

```python
st.info("💡 提示：首次选择关键词后，页面会自动刷新。请切回本页面查看更新后的折线图。")
```

**优点**:
- 无需修改代码逻辑
- 实施成本最低

**缺点**:
- 不解决根本问题
- 用户体验仍有瑕疵

**实施难度**: 🟢 非常简单

---

**推荐方案**:
1. **短期**: 方案 D（添加用户提示）
2. **长期**: 方案 A（使用 `st.fragment`）

**预计工作量**:
- 方案 D: 5分钟
- 方案 B: 30分钟
- 方案 C: 1小时
- 方案 A: 2-3小时

**决策建议**:
- 如果用户反馈影响较小 → 采用方案 D
- 如果用户反馈影响较大 → 采用方案 A
- 如果需要快速修复 → 采用方案 B

## 📊 已知限制

### 限制 #1: Arxiv API 抓取速度

**描述**:
- 抓取大量论文（如5000+篇）时，可能需要较长时间（1-3分钟）
- 受 Arxiv API 的速率限制影响

**影响**:
- 用户需要等待较长时间
- 可能出现超时错误

**缓解措施**:
- 已添加进度提示："正在从Arxiv API抓取数据..."
- 已限制最大查询范围为90天
- 已设置默认 `max_results=10000`

**未来改进**:
- 添加进度条显示抓取进度
- 实施后台抓取 + 缓存机制
- 支持分页加载

---

### 限制 #2: 时间范围限制

**描述**:
- 当前限制查询范围不能超过90天
- 这是为了避免 API 超时和性能问题

**影响**:
- 无法查看超过90天的长期趋势
- 需要分多次查询

**缓解措施**:
- 已从30天放宽到90天
- 在 UI 上显示警告信息

**未来改进**:
- 根据实际性能测试，进一步放宽限制
- 支持自动分段查询 + 合并结果

---

### 限制 #3: 关键词热力图显示限制

**描述**:
- 热力图最多显示20个关键词
- 这是为了避免图表过大、难以阅读

**影响**:
- 如果配置了超过20个关键词，只显示前20个
- 可能遗漏部分关键词的趋势

**缓解措施**:
- 在 UI 上显示提示信息
- 自动选择前20个关键词

**未来改进**:
- 支持用户自定义选择要显示的关键词
- 支持分页显示热力图

---

### 限制 #4: 数据不持久化

**描述**:
- 抓取的数据存储在 `st.session_state` 中
- 刷新页面后数据会丢失，需要重新抓取

**影响**:
- 用户每次访问都需要重新抓取数据
- 无法查看历史数据

**缓解措施**:
- 数据缓存在 session 中，切换 Tab 不会丢失

**未来改进**:
- 实施持久化缓存（Redis/文件缓存）
- 支持导出/导入数据
- 支持历史数据查询
