## 📋 迭代目标

为 PaperBrain 新增图形化 API 设置页面，支持标准提供商（DeepSeek/通义千问/OpenAI 等）和自定义内部 API 两种接入方式，方便开源用户快速配置，同时不影响现有内部 API 使用。

### 核心需求
- 提供图形化 API 配置界面，无需手动编辑 `.env` 文件
- 默认使用标准模式（OpenAI 兼容格式），方便开源用户接入主流 API 提供商
- 保留自定义模式，支持内部部署的 Gemini 格式 API + 自定义 SSL
- 两类 API 模式明确区分，互不影响
- 向后兼容现有 `.env` 配置

## 🏗️ 技术方案

### 架构设计

```
用户打开设置页面
    ↓
选择接入方式
    ├─ 标准模式: 选提供商 → 填 API Key → 选模型
    └─ 自定义模式: 填 URL → 填 Token → 选格式(OpenAI/Gemini)
    ↓
点击"测试连接"
    ↓
点击"保存配置"
    ↓
写入 data/api_config.json
    ↓
热加载服务实例 (reload)
```

### 配置优先级

```
api_config.json (设置页面保存)
    ↓ 不存在时回退
.env 环境变量 (现有配置)
    ↓ 不存在时回退
默认空配置 (未配置状态)
```

### 双角色设计

| 角色 | 用途 | 默认行为 |
|------|------|---------|
| **主 LLM** (`main_llm`) | 论文总结、思维导图、标签、RAG 问答 | 必须配置 |
| **评分 LLM** (`scoring_llm`) | Auto-Scholar 论文评分、PDF 元数据提取 | 默认复用主 LLM |

评分 LLM 可独立配置（如使用豆包），也可复用主 LLM，降低外部用户配置门槛。

## 🔑 关键设计决策

### 1. 为什么用 JSON 文件而非数据库？

- 配置是全局单例，不需要关系查询
- JSON 文件可以手动编辑和备份
- 不需要数据库迁移
- 独立于应用数据，方便管理

### 2. 为什么保留 .env 回退？

- 现有用户（开发者自己）已配置 `.env`，升级后不受影响
- 自动检测 URL 中的 `gemini`/`generateContent` 关键词判断 API 格式
- 自动为 `.env` 来源的配置启用自定义 SSL

### 3. 为什么评分 LLM 默认复用主 LLM？

- 外部用户通常只有一个 API 提供商
- 减少配置步骤：只配一个 API 即可使用全部功能
- 需要独立配置时，勾选复选框即可展开

### 4. 全局服务实例如何处理未配置状态？

- `LLMService.__init__` 中所有属性先赋默认值，再调用 `_load_config()`
- 未配置时 `_configured = False`，服务可导入但不可调用
- 调用 API 时触发 `_ensure_configured()`，给出友好错误提示
- 全局实例创建失败时有 `_create_service()` 安全网兜底

## ⚠️ 注意事项

1. `data/api_config.json` 包含 API Key，已被 `.gitignore` 保护（在 `data/` 目录下）
2. 设置页面保存配置后，服务实例会立即 `reload()`，无需重启应用
3. 自定义 SSL 模式会降低安全级别并跳过证书验证，仅建议内部网络使用
