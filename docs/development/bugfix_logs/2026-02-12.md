# Bug 修复报告 - 2026-02-12

## 修复概述

本次修复解决了 Auto-Scholar 功能中的三个关键问题：
1. "昨天"模式无法抓取论文
2. 评分失败时错误信息不完整
3. 豆包 API SSL 连接失败

## 问题 1：昨天模式无法抓取论文

### 问题描述
- **现象**：选择"昨天"模式点击立即抓取后，流程很快结束，无法抓取任何文章
- **对比**：使用自定义时间段明确设置为昨天的日期时，可以正常抓取 72 篇文章

### 根本原因
在 `ui/auto_scholar.py` 中，"昨天"模式调用 `daily_scheduler.run_daily_pipeline()` 时未传递明确的日期参数：

```python
# 问题代码（第 167-171 行）
else:
    result = daily_scheduler.run_daily_pipeline(
        max_results=500,
        progress_callback=update_ui
    )
```

这导致 `arxiv_crawler.py` 中的日期范围计算不准确。

### 解决方案

#### 1. 重命名功能
将"昨天"改为"昨天到目前"，更准确地描述功能：抓取从昨天 00:00 到今天 23:59 的论文。

#### 2. 修改代码
**文件**：`ui/auto_scholar.py`

```python
# 第 44 行：修改选项名称
fetch_mode = st.selectbox(
    "抓取模式",
    ["昨天到目前", "自定义时间段"],  # 修改：昨天 → 昨天到目前
    key="fetch_mode"
)

# 第 167-176 行：明确传递日期参数
else:  # 昨天到目前模式
    # 昨天 00:00 到今天 23:59
    yesterday = datetime.now() - timedelta(days=1)
    start_dt = datetime.combine(yesterday.date(), datetime.min.time())
    end_dt = datetime.combine(datetime.now().date(), datetime.max.time())
    result = daily_scheduler.run_daily_pipeline(
        max_results=500,
        start_date=start_dt,
        end_date=end_dt,
        progress_callback=update_ui
    )
```

### 验证方法
1. 启动 Streamlit 应用：`streamlit run app.py`
2. 进入 Auto-Scholar 页面
3. 选择"昨天到目前"模式
4. 点击"立即抓取"
5. 验证可以正常抓取论文

## 问题 2：评分失败错误信息不完整

### 问题描述
- **现象**：评分失败时显示：`评分失败（HTTPSConnectionPool(host='aigc.sankuai.com', port=），建议人工复核`
- **问题**：错误信息被截断，无法看到完整的错误原因

### 根本原因
在 `services/scoring_engine.py` 第 74 行，错误信息被截断为前 50 个字符：

```python
# 问题代码
'reason': f'评分失败（{error_msg[:50]}），建议人工复核',
```

### 解决方案

**文件**：`services/scoring_engine.py`

```python
# 第 56-78 行：增强错误处理
except Exception as e:
    error_msg = str(e)
    print(f"❌ 评分失败: {error_msg}")

    # 记录完整的错误堆栈信息
    import traceback
    full_traceback = traceback.format_exc()
    print(f"完整错误堆栈:\n{full_traceback}")

    # 增强的兜底策略
    if "无法从文本中提取有效的 JSON" in error_msg:
        print(f"    🔄 使用兜底评分（JSON 解析失败）")
        return {
            'score': 5.0,
            'reason': '评分系统暂时无法解析 LLM 响应，建议人工复核该论文',
            'title_zh': title,
            'abstract_zh': abstract[:200] + '...' if len(abstract) > 200 else abstract,
            'tags': ['待分类']
        }

    # 显示完整错误信息，不截断
    return {
        'score': 5.0,
        'reason': f'评分失败（{error_msg}），建议人工复核',  # 移除 [:50]
        'title_zh': title,
        'abstract_zh': abstract[:200] + '...' if len(abstract) > 200 else abstract,
        'tags': ['待分类']
    }
```

### 改进效果
- ✅ 显示完整的错误信息
- ✅ 在控制台输出完整的错误堆栈
- ✅ 方便诊断和调试问题

## 问题 3：豆包 API SSL 连接失败

### 问题描述
- **现象**：评分时出现错误：`HTTPSConnectionPool(host='aigc.sankuai.com', port=443): Max retries exceeded`
- **完整错误**：`SSLZeroReturnError: TLS/SSL connection has been closed (EOF) (_ssl.c:1133)`
- **背景**：在本次功能优化之前，豆包 API 可以正常调用

### 诊断过程

#### 1. 创建基础诊断脚本
**文件**：`test_doubao_api.py`

测试结果：
```
✅ 网络连接: 通过 - 可以连接到 aigc.sankuai.com:443
✅ API配置: 通过 - URL 和 Token 已正确配置
❌ API调用: 失败 - SSL/TLS 握手失败
❌ DoubaoService: 失败 - 同样的 SSL 错误
```

#### 2. 创建高级诊断脚本
**文件**：`test_doubao_api_advanced.py`

测试不同的配置方案：
- ✅ 禁用 SSL 验证：调用成功
- ✅ 使用 Session + 自定义 SSL：调用成功
- ✅ 不同的 Header 格式：调用成功

**结论**：豆包 API 的 SSL 证书在 Python requests 库的默认配置下无法通过验证。

### 根本原因

豆包 API 服务器的 SSL/TLS 配置与 Python requests 库的默认 SSL 验证策略不兼容，导致握手失败。

**环境信息**：
- Python 版本: 3.9.19
- requests 版本: 2.31.0
- urllib3 版本: 1.26.19
- OpenSSL 版本: OpenSSL 3.0.14

### 解决方案

**文件**：`services/doubao_service.py`

#### 1. 添加自定义 SSL 适配器

```python
import urllib3
from requests.adapters import HTTPAdapter
from urllib3.util.ssl_ import create_urllib3_context

# 禁用 SSL 警告
urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

class SSLAdapter(HTTPAdapter):
    """自定义 SSL 适配器，解决豆包 API 的 SSL 证书验证问题"""

    def init_poolmanager(self, *args, **kwargs):
        """初始化连接池管理器，使用宽松的 SSL 配置"""
        context = create_urllib3_context()
        # 降低 SSL 安全级别以兼容豆包 API
        context.set_ciphers('DEFAULT@SECLEVEL=1')
        kwargs['ssl_context'] = context
        return super().init_poolmanager(*args, **kwargs)
```

#### 2. 修改 DoubaoService 类

```python
class DoubaoService:
    def __init__(self):
        self.api_url = config.DOUBAO_API_URL
        self.bearer_token = config.DOUBAO_BEARER_TOKEN

        if not self.api_url or not self.bearer_token:
            raise ValueError("豆包 API 配置未设置...")

        # 创建一个持久的 Session，使用自定义 SSL 配置
        self.session = requests.Session()
        self.session.mount('https://', SSLAdapter())
        # 禁用 SSL 验证（因为豆包 API 的证书有问题）
        self.session.verify = False

    def _call_api(self, prompt: str, temperature: float = 0.3,
                  max_tokens: int = 4096) -> str:
        # ... 其他代码 ...

        # 使用 Session 对象发送请求（已配置自定义 SSL）
        response = self.session.post(
            self.api_url,
            headers=headers,
            data=json_data,
            timeout=120
        )
        # ... 其他代码 ...
```

### 技术原理

1. **SECLEVEL=1**：降低 OpenSSL 的安全级别，允许使用较旧的加密算法
2. **verify=False**：禁用 SSL 证书验证（适用于内部 API）
3. **Session 对象**：复用连接，提高性能

### 安全性说明

⚠️ **注意**：本次修复禁用了豆包 API 的 SSL 证书验证。

**适用场景**：
- ✅ 内部 API 或受信任的环境
- ✅ 开发和测试环境
- ✅ 已知安全的网络环境

**不适用场景**：
- ❌ 公网环境的生产系统
- ❌ 处理敏感数据的场景
- ❌ 需要严格安全审计的环境

**建议**：
- 如果需要更高的安全性，请联系豆包 API 技术支持解决证书问题
- 或者在生产环境中配置正确的 SSL 证书
