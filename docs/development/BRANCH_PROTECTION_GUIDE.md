# 设置仓库为私密的步骤

## 方法：将整个仓库设为私密

### 步骤：

1. **访问仓库设置**
   - 打开：https://github.com/cr-bh/Paper-Brain
   - 点击 **Settings**（设置）

2. **修改可见性**
   - 滚动到页面最底部的 **Danger Zone**（危险区域）
   - 找到 **Change repository visibility**（更改仓库可见性）
   - 点击 **Change visibility**

3. **选择私密**
   - 选择 **Make private**（设为私密）
   - 输入仓库名称确认：`cr-bh/Paper-Brain`
   - 点击确认

### 效果：

- ✅ 整个仓库（包括所有分支）只有您能访问
- ✅ `main` 和 `main-private` 都受保护
- ✅ 不需要设置分支保护规则
- ❌ 其他人无法查看您的公开代码

---

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **不设置保护** | 简单，您可以自由推送 | 依赖 GitHub 默认权限 | 个人仓库，不需要额外保护 |
| **私密仓库** | 最安全，所有内容都私密 | 无法公开展示项目 | 完全私密的项目 |
| **Lock branch** | 防止意外修改 | 您自己也无法推送 | 归档的分支 |

## 我的建议

### 如果您想公开展示项目：

**保持当前设置**：
- `main` 分支：公开，API 已脱敏
- `main-private` 分支：在同一个公开仓库中，但不设置保护
- 只有您有推送权限，已经足够安全

### 如果您不需要公开展示：

**设置为私密仓库**：
- 整个仓库私密
- 所有分支都只有您能访问
- 最安全的方案
